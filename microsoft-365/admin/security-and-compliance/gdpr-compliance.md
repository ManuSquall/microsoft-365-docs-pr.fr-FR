# <a name="gdpr-simplified-a-guide-for-your-small-business"></a>RGPD simplifié : Guide pour votre petite entreprise

 *Utilisation de Microsoft 365 pour les entreprises afin d’atténuer et de gérer la conformité RGPD* 
  
Le règlement général sur la protection des données (RGPD) est une réglementation de l’Union européenne (UE) qui stipule comment une organisation doit gérer les données personnelles. Le RGPD est dû à l’entrée en vigueur le 25 mai 2018. Si votre entreprise vend à, fournit des services ou utilise des citoyens de l’Union européenne, le [RGPD](https://ec.europa.eu/info/law/law-topic/data-protection/reform/what-does-general-data-protection-regulation-gdpr-govern_en) vous affectera. 
  
Cet article vous aide à comprendre ce qu’est la RGPD, pourquoi elle a été fournie et comment Microsoft 365 pour les entreprises peut aider votre organisation à se conformer à la RGPD.
  
## <a name="a-quick-overview-of-the-gdpr"></a>Vue d’ensemble rapide du RGPD

Le RGPD est une réglementation de l’Union européenne qui met à jour et étend la directive de protection des données antérieure (DPD) en premier lieu dans 1995. Le RGPD est concerné par la confidentialité des données d’un individu, il s’agit d’un client, d’un client, d’un employé ou d’un partenaire commercial. L’objectif du RGPD est de renforcer la protection des données personnelles pour les citoyens de l’UE, qu’ils se trouvent dans l’Union européenne ou ailleurs. La réglementation définit les attentes et indique comment y parvenir. Lorsque le RGPD est exécutoire à la fin du 2018 mai, les organisations doivent disposer de mesures en place qui satisfont aux exigences de l’RGPD.
  
Le RGPD concerne toutes les données et la manière dont elles sont utilisées. Les données sont considérées comme ayant un cycle de vie. Le cycle démarre lorsque vous collectez des données, continue lorsque vous les stockez et que vous les utilisez (traitement) et se termine lorsque vous les supprimez complètement de vos systèmes. 
  
Le RGPD est concerné par les types de données suivants : 
  
- **Données personnelles :** Si vous pouvez lier des données à un individu et les identifier, ces données sont considérées comme personnelles par rapport au RGPD. Des exemples de données personnelles incluent le nom, l’adresse, la date de naissance et l’adresse IP. Le RGPD considère les informations codées (également appelées « pseudonymes ») comme des données personnelles, quelle que soit la manière dont les données sont masquées ou techniques, si les données peuvent être liées à un individu.
    
- **Données personnelles sensibles** Il s’agit de données qui ajoutent des détails supplémentaires aux données personnelles. Exemples : religion, appartenance syndicale, origine ethnique, etc. Les données personnelles sensibles incluent également les données biométriques et l’ADN. Sous RGPD, les données sensibles ont des règles de protection plus rigoureuses que les données personnelles. 
    
## <a name="gdpr-terms"></a>Conditions RGPD

Vous verrez quelques termes fréquemment cités dans le RGPD. Il est important de comprendre ces termes.
  
 **Consentement**
  
États RGPD : « le traitement des données personnelles doit être conçu pour accueillir l’humanité ». Le RGPD espère atteindre cet objectif en utilisant le consentement lors du traitement des données personnelles. Cela peut être le simple acte de demander à vos clients s’ils souhaitent recevoir des messages électroniques de votre entreprise. Il ne s’agit pas non plus de cases à cocher d’exclusion sur votre site Web lorsque vous souhaitez utiliser des données pour le marketing. Vous devez avoir un consentement explicite à l’aide d’un « acte affirmatif ». Par ailleurs, vous devez également conserver des enregistrements relatifs à la prise ou à la révocation d’un consentement.
  
 **Droits des personnes concernées**
  
Le RGPD établit les droits des personnes concernées, ce qui signifie que, en ce qui concerne les données personnelles, les clients, les employés, les partenaires professionnels, les clients, les entrepreneurs, les fournisseurs, etc., ont le droit d’effectuer les opérations suivantes :
  
- **Être informé de leurs données :** Vous devez informer les utilisateurs de votre utilisation de leurs données. 
    
- **Avoir accès à leurs données :** Vous devez accorder à des utilisateurs l’accès à toutes les données que vous conservez (par exemple, à l’aide de l’accès aux comptes ou de manière manuelle). 
    
- **Demande de rectification de données :** Les utilisateurs peuvent vous demander de corriger les données inexactes. 
    
- **Demander la suppression des données :** Également appelé « droit d’effacement », ce droit permet à un individu de demander à ce qu’une de ses données personnelles collectées par une société soit supprimée sur tous les systèmes qui l’utilisent ou les partagent. 
    
- **Traitement restreint des demandes :** Une personne peut vous demander de supprimer ou de restreindre ses données. Toutefois, il s’applique uniquement dans certaines circonstances. 
    
- La **portabilité des données est la suivante :** Une personne peut demander à ce que ses données soient transférées à une autre société. 
    
- **Objet :** Une personne peut utiliser les données utilisées pour différentes utilisations, y compris le marketing direct. 
    
- **Demander à ne pas être soumis à une prise de décision automatisée, y compris le profilage :** Le RGPD a des règles strictes quant à l’utilisation des données pour profiler des personnes et à automatiser les décisions basées sur ce profilage. 
    
## <a name="common-small-business-questions-about-the-gdpr"></a>Questions courantes sur les petites entreprises concernant le RGPD

### <a name="im-a-sole-proprietor---do-i-really-have-to-worry-about-the-gdpr"></a>Je suis un seul propriétaire : est-ce que je dois vraiment me soucier de la RGPD ?

Le RGPD concerne les données que vous traitez, pas le nombre d’employés dont vous disposez. Elle affecte les sociétés de toutes tailles, même les propriétaires uniques. Toutefois, les entreprises de moins de 250 employés disposent de certaines exemptions, telles que la conservation des enregistrements, mais uniquement si vous êtes certain que le traitement des données n’affecte pas les droits de l’utilisateur et qu’il est traité occasionnellement.
  
Par exemple, le traitement des données non personnelles serait exempté ou nécessite des mesures réduites. Toutefois, si vous traitez des données qui sont considérées comme des « données sensibles de catégorie spéciale », même si ce n’est qu’occasionnellement, vous devrez enregistrer ce traitement de données. La définition de « traitement occasionnel » est vague, mais elle est destinée à s’appliquer à des données qui sont utilisées une seule fois ou rarement.
  
Vous devez également vous assurer que les données personnelles que vous collectez sont protégées. Cela signifie que vous devez la chiffrer et vous assurer que l’accès à celle-ci est contrôlé à l’aide d’un mot de passe au moins. La conservation des données de votre client dans une feuille de calcul sur votre ordinateur de bureau sans protection ne répondra pas aux attentes RGPD. 
  
### <a name="how-can-i-tell-if-our-company-website-is-gdpr-compliant"></a>Comment puis-je savoir si le site Web de notre entreprise est compatible RGPD ?

La première question à se poser est la suivante : souhaitez-vous recueillir des données personnelles n’importe où sur votre site ? Par exemple, vous pouvez avoir un formulaire de contact qui vous demande un nom et une adresse de messagerie. Si vous souhaitez envoyer des courriers électroniques marketing, veillez à ajouter une case à cocher « opt-in » qui explique exactement ce que vous utiliserez pour les données. Uniquement si le destinataire coche cette zone, vous pouvez utiliser ses données personnelles à des fins de marketing.
  
Vérifiez également que la base de données qui stocke les données est protégée. Votre société d’hébergement Web ou votre fournisseur de stockage cloud pourra vous en informer. Si vous utilisez Microsoft 365 pour les entreprises, le stockage des données est conforme à la norme RGPD. 
  
### <a name="my-company-is-outside-europe-does-the-gdpr-really-affect-us"></a>Ma société se trouve en dehors de l’Europe. Le RGPD affecte-t-il réellement ?

Le RGPD est une réglementation qui protège les citoyens de l’UE. Si votre entreprise gère les citoyens de l’UE maintenant, ou si vous l’espérez à l’avenir, vous en serez affecté. Cela s’applique aux citoyens vivant dans l’état de l’Union européenne et ceux vivant ailleurs.
  
Examinez les exemples suivants :
  
- Une société américaine qui loue des voitures aux citoyens de l’UE doit satisfaire aux exigences RGPD lors de la collecte et du traitement des données du client. La société devra accepter le consentement lorsqu’elle prend les données du client et s’assurer que les données sont stockées de manière sécurisée. Ils devront également s’assurer que le client peut appliquer tous leurs droits de personne. 
    
- Une société australienne vend des produits en ligne et ses utilisateurs configurent des comptes en ligne. RGPD les droits des personnes concernées et le consentement seront appliqués aux citoyens de l’Union européenne qui ouvrent un compte. La société devra s’assurer que le client peut appliquer tous ses droits de personne. 
    
- Une œuvre caritative internationale recueille des données sur les donateurs et les utilise pour envoyer des mises à jour et des demandes de dons. États RGPD : '... le traitement des données personnelles à des fins de marketing direct peut être considéré comme effectué pour un intérêt légitime. " Toutefois, la responsabilité de l’organisation est de prouver que leurs intérêts remplacent ceux de la personne concernée par les données. La société (ou dans le cas présent, l’organisation charitable) doit toujours recevoir une autorisation explicite, explicite et d’adhésion.
    
Le RGPD s’applique également si les données du client se déplacent dans les bordures. Si vous utilisez le Cloud Computing pour le stockage de données, vous devez vous assurer que le service est entièrement compatible RGPD. Cela peut être compliqué si le stockage de données se trouve dans des emplacements qui ont un enregistrement médiocre de la protection des données. Si vous utilisez Microsoft 365 pour les entreprises, nous disposons de la documentation juridique correcte en place pour répondre aux exigences de RGPD.
  
### <a name="sure-i-collect-data-but-some-other-company-stores-it-does-that-get-me-off-the-hook"></a>Bien sûr, j’ai collecté des données, mais une autre société les stocke. Faut-il m’arrêter ?

Sous RGPD, si vous collectez des données que vous êtes affectées à une certaine étendue. Le RGPD a le concept de processeur de données et de contrôleur de données : 
  
- **Contrôleur de données :** Une personne ou une organisation (vous pouvez avoir des contrôleurs communs) qui déterminent la manière dont les données sont collectées. Ils peuvent le stocker en utilisant les serveurs Cloud d’une autre entreprise. Par exemple, un site Web qui recueille les données client est un contrôleur. 
    
- **Processeur de données :** Personne ou organisation qui stocke des données pour le compte du (des) contrôleur (s) et traite ces données à la demande. Par exemple, les applications Microsoft 365 pour le stockage de données métiers agissent comme un processeur et sont entièrement conformes à RGPD. 
    
    Une organisation ou un système peut agir comme un contrôleur et un processeur. Microsoft 365 pour entreprise peut agir en tant que et est conforme à la RGPD.
    
### <a name="can-i-still-send-out-marketing-emails-to-my-old-customers"></a>Puis-je toujours envoyer des courriers électroniques marketing à mes anciens clients ?

Vous devez vous assurer que vos clients, y compris ceux que vous avez effectués depuis des années, ont consenti à utiliser leurs données pour le marketing. Vous avez peut-être déjà capturé le consentement, ainsi qu’un enregistrement pour l’afficher. Si c’est le cas, tous les sont configurés pour continuer le marketing. Si ce n’est pas le cas, vous devez obtenir l’autorisation du client pour continuer à le commercialiser. Cela implique généralement l’envoi d’un message électronique demandant aux clients d’accéder à votre site et de sélectionner une option pour recevoir des courriers futurs. 
  
### <a name="do-i-have-to-worry-about-the-gdpr-when-i-recruit-new-employees-what-about-current-employees"></a>Dois-je me préoccuper de l’RGPD lorsque je recrute de nouveaux employés ? Qu’en est-il des employés actuels ?

Le RGPD n’affecte pas simplement les données client ; Il s’étend également aux données des employés. Les nouveaux recruteurs se trouvent souvent à l’aide de plateformes de réseaux sociaux telles que LinkedIn. Assurez-vous que vous ne stockez pas de données de recrutement potentielles sans leur autorisation expresse.
  
En ce qui concerne les employés existants et les nouveaux contrats d’employés, une signature à la fin d’un contrat n’assume pas nécessairement le consentement, en particulier lorsqu’une clause non affirmative est utilisée dans un contrat. Dans ce cas, vous devez capturer le consentement de manière explicite associée à la clause. Cela signifie que dépend de votre contrat employé, mais vous pouvez utiliser des « intérêts légitimes » dans certains cas et ajouter une notification de traitement des données de l’employé pour vous assurer que vos employés connaissent bien ce que vous allez faire avec leurs données.
  
## <a name="satisfy-privacy-concerns-using-microsoft-365-for-business"></a>Répondre aux préoccupations de confidentialité à l’aide de Microsoft 365 pour les entreprises

La conformité avec le RGPD consiste à s’assurer que les données personnelles sont protégées. Le RGPD a un concept connu sous le nom de confidentialité par conception et par défaut. Cela signifie que la protection des données doit être « intégrée » à un système et à un produit afin que les préoccupations de confidentialité soient la deuxième nature. 
  
À l’instar de leurs plus grandes entreprises, une petite entreprise a besoin de commodité sans sacrifier la sécurité. Microsoft 365 for Business est conçu pour les entreprises de moins de 300 employés. Les petites entreprises peuvent utiliser des outils Microsoft Cloud pour améliorer la productivité de l’entreprise. Avec Microsoft 365 for Business, une petite entreprise peut gérer des courriers électroniques, de la documentation, voire des réunions et des événements. Elle comporte également des mesures de sécurité intégrées et la gestion des périphériques, qui sont essentielles pour la conformité RGPD.
  
Microsoft 365 pour les entreprises peut vous aider avec le processus RGPD des façons suivantes :
  
- **Découvrez :** Une étape importante de la conformité RGPD est de savoir quelles données vous avez. 
    
- **Gérer :** Le contrôle de l’accès aux données et la gestion de leur utilisation font partie intégrante de RGPD. Microsoft 365 pour entreprises protège les données métiers en fonction des stratégies que vous souhaitez appliquer aux appareils. La gestion des appareils est essentielle dans un âge où les employés travaillent à distance. Microsoft 365 for Business inclut des fonctionnalités de gestion des appareils qui assurent la protection des données sur tous les appareils. Par exemple, vous pouvez spécifier que tous les appareils Windows 10 de votre entreprise sont protégés via Windows Defender. 
    
- **Protéger :** Microsoft 365 pour les entreprises est conçu pour la sécurité. Ses contrôles de gestion des périphériques et de protection des données fonctionnent sur votre réseau d’entreprise, y compris les appareils distants, pour garantir la sécurité des données. Microsoft 365 pour les entreprises offre des contrôles tels que les paramètres de confidentialité dans les applications Office et le chiffrement des documents. Avec Microsoft 365 pour les entreprises, vous pouvez effectuer une surveillance de la conformité RGPD pour vous assurer que le niveau de protection approprié est défini. 
    
- **Rapport :** Le RGPD met tout en évidence sur la création de rapports. Même une entreprise avec un seul employé, si cette activité traite de grandes quantités de données, il est nécessaire de documenter et de rendre compte de leurs procédures. Microsoft 365 pour les entreprises prend le temps de Reporting requis pour les organisations de plus petite taille. 
    
    Des outils tels que les journaux d’audit vous permettent de suivre et de signaler le déplacement de données. Les rapports incluent la classification des données collectées et stockées, ce que vous faites avec les données et les transferts de données. 
    
Les clients, les employés et les clients sont de plus en plus conscients de l’importance de la confidentialité des données et s’attendent à présent à ce qu’une société ou une organisation respecte cette confidentialité. Microsoft 365 for Business vous fournit les outils permettant d’atteindre et de maintenir la conformité RGPD sans une Upheaval importante à votre entreprise.
