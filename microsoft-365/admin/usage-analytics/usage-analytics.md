# <a name="microsoft-365-usage-analytics"></a>Analyse de l'utilisation de Microsoft 365

Utilisez Microsoft 365'utilisation de l’Power BI pour obtenir des informations sur la façon dont votre organisation adopte les différents services au sein Microsoft 365. Vous pouvez visualiser et analyser les données Microsoft 365'utilisation, créer des rapports personnalisés et partager les informations au sein de votre organisation. Vous pouvez également avoir un aperçu de la façon dont des régions ou des départements spécifiques utilisent Microsoft 365.
  
Microsoft 365'analyse de l’utilisation vous donne accès à un tableau de bord pré-construit qui fournit une vue croisée des produits des 12 derniers mois et contient un certain nombre de rapports pré-construits. Chaque rapport fournit des informations spécifiques sur l'utilisation. Des informations spécifiques à l’utilisateur sont disponibles pour le dernier mois civil complet.
  
Le [modèle de données](usage-analytics-data-model.md) qui alimente l’application modèle inclut les attributs utilisateur d’Active Directory, ce qui permet de pivoter dans certains rapports. Les attributs suivants de l’annuaire actif sont inclus : emplacement, département et organisation. 
  
Pour commencer à collecter des données, voir [Activer l'analyse de l'utilisation de Microsoft 365](enable-usage-analytics.md). 
  
Microsoft 365'utilisation des données contient un certain nombre de rapports détaillés dans les sections suivantes. 

Vous pouvez accéder à des rapports détaillés pour chaque zone en sélectionnant les tableaux de données. Vous pouvez afficher tous les rapports pré-construits en sélectionnant les onglets en bas du site. Pour des instructions plus détaillées, [lisez Naviguer et utiliser les rapports et](navigate-and-utilize-reports.md) personnaliser les [rapports](customize-reports.md).

## <a name="executive-summary"></a>Résumé

Le résumé est une vue d’ensemble de haut niveau de l’Microsoft 365 pour l’adoption, l’utilisation, la mobilité, la communication, la collaboration et les rapports de stockage des entreprises, et s’adresse aux décideurs d’affaires. Il donne un aperçu de la façon dont certains services individuels sont utilisés, en fonction de tous les utilisateurs qui ont été activés et ceux qui sont actifs. Toutes les valeurs du mois indiquées dans le rapport se réfèrent au dernier mois complet. 

Ce résumé vous permet de comprendre rapidement les habitudes d’utilisation Office et comment et où vos employés collaborent.

![Image du résumé de Microsoft 365'utilisation des produits.](../../media/office365usage-exec-summary.png)

## <a name="overview"></a>Présentation

Le Microsoft 365 d’aperçu contient les rapports suivants. Vous pouvez les afficher en choisissant l’onglet en haut de la page de rapport. Toutes les valeurs du mois indiquées dans la section supérieure du rapport se réfèrent au dernier mois complet.

- **Adoption** &ndash; Offre un résumé complet des tendances en matière d’adoption. Utilisez les rapports de cette section pour savoir comment vos utilisateurs ont adopté des Microsoft 365, ainsi que comment l’utilisation globale des services individuels a changé d’un mois à l’autre. Vous pouvez voir comment les utilisateurs peuvent être activés, combien de personnes dans votre organisation utilisent activement Microsoft 365, combien sont des utilisateurs de retour, et combien utilisent le produit pour la première fois.

- **Utilisation** &ndash; Offre une vue d’ensemble du volume d’utilisateurs actifs et des activités clés pour chaque produit au cours des 12 derniers mois. Utilisez les rapports de cette section pour savoir comment les membres de votre organisation utilisent Microsoft 365.

- **Communication** &ndash; Vous pouvez voir en un coup d’œil si les gens de votre organisation préfèrent rester en contact en utilisant des Teams, Yammer, des courriels ou des appels Skype appels. Vous pouvez observer s’il y a des changements dans les modèles dans l’utilisation des outils de communication parmi vos employés. 

- **Collaboration** &ndash; Découvrez comment les membres de votre organisation utilisent les OneDrive et les SharePoint pour stocker des documents et collaborer les uns avec les autres, et comment ces tendances évoluent d’un mois à l’autre. Vous pouvez également voir combien de documents sont partagés à l’interne ou à l’externe et combien de sites SharePoint ou comptes OneDrive sont activement utilisés, éclatés par les propriétaires et autres collaborateurs.

- **Stockage** &ndash; Utilisez ce rapport pour suivre le stockage dans le cloud pour les boîtes aux lettres, OneDrive et les sites SharePoint cloud.

- **Mobilité** &ndash; Suivez les clients et les appareils que les gens utilisent pour se connecter au courrier électronique, Teams, Skype ou aux Yammer.

## <a name="activation-and-licensing"></a>Activation et licence

La page d’activation et de licence offre des rapports sur Microsoft 365 activation; c’est-à-dire le nombre d’utilisateurs téléchargés et Office applications et le nombre de licences attribuées par votre organisation. La valeur du mois vers le haut se réfère au mois en cours, et les mesures reflètent les valeurs agrégées du début du mois à la date actuelle.

- **Activation** &ndash; Suivez les activations de plan de service (par exemple, Applications Microsoft 365 pour les grandes entreprises, Project et Visio) dans votre organisation. Chaque personne disposant d'une licence Office peut installer les applications sur jusqu'à cinq appareils. Vous pouvez également utiliser des rapports dans cette section pour voir les appareils sur lesquels les gens ont installé Office applications. Notez que pour activer un plan, un utilisateur doit installer l’application et se connecter avec son compte.

- **Octroi de licences** &ndash; Ce rapport contient un aperçu des types de licences, du nombre d’utilisateurs qui ont été affectés à chaque type de licence et de la distribution des attributions de licences pour chaque mois. La valeur du mois vers le haut se réfère au mois en cours, et les mesures reflètent les valeurs agrégées du début du mois à la date actuelle.

## <a name="product-usage"></a>Utilisation du produit

Ce rapport contient un rapport distinct pour chaque service Microsoft 365, y compris les Exchange, les groupes Microsoft 365, OneDrive, SharePoint, Skype, Teams et Yammer. Chaque rapport contient des rapports d’utilisateurs actifs totaux par rapport au total des utilisateurs actifs, des comptes d’entités telles que des boîtes aux lettres, des sites, des groupes et des comptes, ainsi que des rapports de type activité, le cas échéant. Toutes les valeurs du mois indiquées dans la section supérieure du rapport se réfèrent au dernier mois complet.

## <a name="user-activity"></a>Activité utilisateur

Les rapports d’activité des utilisateurs sont disponibles pour certains services individuels. Ces rapports fournissent des données d’utilisation des détails au niveau de l’utilisateur jointes aux attributs actives de l’annuaire. De plus, le rapport d’adoption du Ministère vous permet de trancher par attributs d’annuaire actif afin que vous puissiez voir les utilisateurs actifs dans tous les services individuels. Toutes les mesures sont agrégées pour le dernier mois complet.

## <a name="faq"></a>FAQ

### <a name="is-this-template-app-going-to-be-available-through-purchase-or-will-it-be-free"></a>Est-ce que cette application modèle va être disponible à l’achat ou sera-t-elle gratuite ?

Il n’est pas gratuit, vous aurez besoin d’Power BI Pro licence. Pour plus de [détails, consultez les](/power-bi/service-template-apps-install-distribute#prerequisites) conditions préalables à l’installation, à la personnalisation et à la distribution d’une application modèle.

Pour partager les tableaux de bord avec d’autres, veuillez en voir plus sur [les tableaux de bord et les rapports Share](/power-bi/service-how-to-collaborate-distribute-dashboards-reports#share-dashboards-and-reports).
### <a name="is-the-usage-summary-reports-reader-role-enough-to-view-the-usage-analytics"></a>Le rôle du lecteur de rapports sommaires d’utilisation est-il suffisant pour afficher l’analyse de l’utilisation ?

Le rôle De lecteur de rapports sommaires d’utilisation ne permet l’accès aux agrégats de niveau locataire que dans Microsoft 365'utilisation.  Nous recommandons le rôle de lecteur de rapports ou de rapports sommaires d’utilisation lecteur à toute personne responsable de la gestion du changement et de l’adoption, mais n’est pas nécessairement un administrateur informatique.

### <a name="who-can-connect-to-microsoft-365-usage-analytics"></a>Qui peut se connecter aux analyses de l'utilisation de Microsoft 365 ?

Vous devez être soit un **administrateur global**, **administrateur Exchange**, **administrateur Skype Entreprise**, administrateur **SharePoint**, lecteur **global ou** lecteur **de rapport** afin d’établir la connexion à l’application modèle. Voir Sur [les rôles admin pour](../add-users/about-admin-roles.md) plus d’informations.

### <a name="who-can-customize-the-usage-analytics-reports"></a>Qui personnaliser les rapports d’analyse d’utilisation ?

Seul l’utilisateur qui a fait la connexion initiale à l’application modèle peut personnaliser les rapports ou créer de nouveaux rapports dans l’interface Power BI web. Voir [Personnaliser les rapports dans l’analyse Microsoft 365'utilisation des](customize-reports.md) instructions.

### <a name="can-i-only-customize-the-reports-from-the-power-bi-web-interface"></a>Puis-je uniquement personnaliser les rapports à partir de l Power BI interface Web ?

En plus de personnaliser les rapports à partir de l’interface Web Power BI, les utilisateurs peuvent également utiliser des Power BI Desktop pour se connecter directement au service de reporting Microsoft 365 pour construire leurs propres rapports.

### <a name="how-can-i-get-the-pbit-file-that-this-dashboard-is-associated-with"></a>Comment obtenir le fichier pbit auquel ce tableau de bord est associé ?

Vous pouvez accéder au fichier pbit à partir du [centre de téléchargement Microsoft](https://download.microsoft.com/download/7/8/2/782ba8a7-8d89-4958-a315-dab04c3b620c/Microsoft%20365%20Usage%20Analytics.pbit).

### <a name="who-can-view-the-dashboards-and-reports"></a>Qui pouvez voir les tableaux de bord et les rapports?

Si vous vous êtes connecté à l’application modèle, vous pouvez la partager avec n’importe qui en utilisant [la fonctionnalité de partage](/power-bi/collaborate-share/service-share-dashboards). Power BI licence exige que le partage de l’utilisateur et l’utilisateur avec lequel un tableau de bord est partagé aient Power BI Pro ou Power BI Premium.

### <a name="can-anyone-share-the-dashboard-or-does-it-have-to-be-the-person-who-connected-to-the-dashboard"></a>N'importe qui peut-il partager le tableau de bord ou doit-il s'agir obligatoirement de la personne qui s'est connectée au tableau de bord ?

Lorsque vous partagez le tableau de bord, vous pouvez soit permettre aux utilisateurs de re-partager le tableau de bord avec d’autres ou non. Vous pouvez définir cette option au moment du partage.

### <a name="is-it-possible-to-work-on-and-customize-the-same-template-app-with-a-group-of-people"></a>Est-il possible de travailler et de personnaliser la même application modèle avec un groupe de personnes ?

Oui. Pour permettre à un groupe d’administrateurs de travailler ensemble sur la même application de modèle, vous pouvez tirer parti des fonctionnalités de l’espace de travail de l’application de Power BI, pour plus d’informations, [voir Comment dois-je collaborer et partager des tableaux de bord et des rapports?](/power-bi/collaborate-share/service-how-to-collaborate-distribute-dashboards-reports) 

### <a name="for-which-timeframe-is-data-available"></a>Pour quelle période des données sont-elles disponibles ?

La majorité des rapports affichent des données pour les 12 mois précédents. Toutefois, certains graphiques peuvent montrer moins d’historique puisque la collecte de données pour différents produits et rapports ont été commencés à des moments différents et donc les données pour les 12 mois complets pourraient ne pas être disponibles. Tous les rapports finiront par accumuler jusqu’à 12 mois d’histoire. Les rapports qui affichent les détails du niveau utilisateur affichent les données pour le mois complet précédent.

### <a name="what-data-is-included-in-the-template-app"></a>Quelles données sont incluses dans l’application modèle ?

Les données de l’application modèle couvrent actuellement le même ensemble de mesures d’activité disponibles dans les rapports [d’activité](../activity-reports/activity-reports.md). Au fur et à mesure que les rapports d’activité sont ajoutés aux rapports d’activité, ils seront ajoutés à l’application modèle dans une version future.

### <a name="how-does-the-data-in-the-template-app-differ-from-the-data-in-the-usage-reports"></a>En quoi les données de l’application modèle diffèrent-elles des données des rapports d’utilisation ?

Les données sous-jacentes que vous voyez dans l’application modèle correspond aux données que vous voyez dans les rapports d’activité dans le Microsoft 365 d’administration. Les principales différences sont que dans les données du centre d’administration est disponible pour les 7/30/90/180 derniers jours tandis que l’application modèle présente des données sur une base mensuelle pour un montant jusqu’à 12 mois.

En outre, les détails au niveau de l’utilisateur dans l’application modèle ne sont disponibles que pour le dernier mois complet pour les utilisateurs qui se sont vu attribuer une licence de produit et ont effectué une activité.

### <a name="when-should-i-use-the-template-app-and-when-the-usage-reports"></a>Quand dois-je utiliser l’application modèle et quand l’utilisation se signale ?

Les [rapports d’activité](../activity-reports/activity-reports.md) sont un bon point de départ pour comprendre l’utilisation et l’adoption Microsoft 365. L’application modèle combine les données d’utilisation Microsoft 365 et les informations actives d’annuaire de votre organisation et permet aux administrateurs d’analyser l’ensemble de données en utilisant les capacités d’analyse visuelle de Power BI. Cela permet aux administrateurs non seulement de visualiser et d’analyser les données d’utilisation Microsoft 365, mais aussi de les trancher par des propriétés actives d’annuaire telles que les départements, l’emplacement, etc. Ils peuvent également créer des rapports personnalisés et partager les informations au sein de leur organisation. 

### <a name="how-often-is-the-data-refreshed"></a>À quelle fréquence les données sont-elles actualisées ? 

Lorsque vous vous connectez à l’application modèle pour la première fois, elle se remplira automatiquement de vos données pour les 12 mois précédents. Après cela, les données de l’application modèle se rafraîchira chaque semaine. Les clients peuvent choisir de modifier l’horaire de rafraîchissement si leur utilisation de ces données exige un rythme de mise à jour différent.

Le service de Microsoft 365 arrière actualisera les données sur une base quotidienne et fournit des données qui se situent entre 5 et 8 jours latents par rapport à la date actuelle.

La **colonne date** de contenu dans chaque ensemble de données représente la date de fraîcheur des données de l’application modèle.

### <a name="how-is-an-active-user-defined"></a>Comment un utilisateur actif est-il défini ?

La définition d’utilisateur actif est la même que la définition [d’utilisateur actif dans](../activity-reports/active-users.md) les rapports d’activité.

### <a name="what-sharepoint-site-collections-are-included-in-the-sharepoint-reports"></a>Quelles collections de sites SharePoint sont incluses dans les rapports SharePoint ?

La version actuelle de l’application modèle inclut l’activité des fichiers à partir SharePoint sites d’équipe et SharePoint sites de groupe.

### <a name="which-groups-are-included-in-the-microsoft-365-groups-usage-report"></a>Quels groupes sont inclus dans le rapport d’utilisation Microsoft 365 groupes de défense des groupes?

La version actuelle de l’application modèle inclut l’utilisation Outlook groupes, Yammer groupes et SharePoint groupes. Il n’inclut pas les groupes liés à Microsoft Teams ou planificateur.

### <a name="when-will-an-updated-version-of-the-template-app-become-available"></a>Quand une version mise à jour de l’application modèle sera-t-elle disponible ?

Des modifications majeures à l’application modèle seront publiées deux fois par an, ce qui peut inclure de nouveaux rapports ou de nouvelles données. Des modifications mineures aux rapports peuvent être publiées plus fréquemment.

### <a name="is-it-possible-to-integrate-the-data-from-the-template-app-into-existing-solutions"></a>Est-il possible d’intégrer les données de l’application modèle dans les solutions existantes ? 

Les données de l’application modèle peuvent être récupérées par l’intermédiaire Microsoft 365 API (en avant-première). Lorsqu’ils expédions à la production, ils seront [fusionnés au sein de la Graph Microsoft et les API de déclaration](https://go.microsoft.com/fwlink/p/?linkid=848843). 

### <a name="are-there-plans-to-expand-the-template-app-to-show-usage-data-from-other-microsoft-products"></a>Envisagez-vous d’étendre l’application modèle pour afficher les données d’utilisation d’autres produits Microsoft ?

Ceci est envisagé pour des améliorations futures. Consultez la feuille [de Microsoft 365 pour les](https://www.microsoft.com/microsoft-365/roadmap) mises à jour.

### <a name="how-can-i-pivot-by-company-information-in-active-directory"></a>Comment puis-je pivoter par informations société dans Active Directory ?

Les informations de l’entreprise sont incluses dans l’un des champs d’annuaire actif dans l’application modèle et vous pouvez les voir comme un filtre pré-construit dans les rapports **d’activité de l’utilisateur du** produit. Il est disponible en tant que colonne dans la table **UserState.**

### <a name="is-it-possible-to-bring-in-additional-fields-from-active-directory"></a>Est-il possible d'importer des champs supplémentaires à partir d'Active Directory ?

Une personnalisation supplémentaire sur ces données est possible en se connectant [aux API de rapports Microsoft Graph pour](https://go.microsoft.com/fwlink/p/?linkid=848843) extraire des champs supplémentaires de Azure Active Directory et rejoindre l’ensemble de données. 

### <a name="is-it-possible-to-aggregate-the-information-in-the-template-app-across-multiple-subscriptions"></a>Est-il possible d’agréger les informations dans l’application modèle sur plusieurs abonnements ?

À l’heure actuelle, l’application modèle est pour un seul abonnement, car elle est associée aux informations d’identification qui ont été utilisées pour se connecter initialement à elle.

### <a name="is-it-possible-to-see-usage-by-plan-ie-e1-e3"></a>Est-il possible de voir l’utilisation par plan (c.-à-d. E1, E3)?

Dans l’application modèle, l’utilisation est représentée au niveau par produit. Les données sur les différents abonnements attribués aux utilisateurs sont fournies, mais il n’est pas possible de corréler l’activité de l’utilisateur à l’abonnement attribué à l’utilisateur.

### <a name="is-it-possible-to-integrate-other-data-sets-into-the-template-app"></a>Est-il possible d’intégrer d’autres ensembles de données dans l’application modèle ?

Vous pouvez utiliser Power BI Desktop pour vous connecter aux API Microsoft 365 (en avant-première) pour apporter des sources de données supplémentaires à combiner avec les données de l’application modèle.

Pour plus d’informations, consultez [le document Personnaliser](customize-reports.md).

### <a name="is-it-possible-to-see-the-top-users-reports-for-a-specific-timeframe"></a>Est-il possible de voir les rapports « Meilleurs utilisateurs » pour une période spécifique?

Tous les rapports au niveau des utilisateurs présentent des données agrégées pour le mois précédent.

### <a name="will-the-template-app-be-localized"></a>L’application modèle sera-t-elle localisée ? 

Ce n’est actuellement pas sur la feuille de route.

### <a name="i-have-a-specific-question-about-the-data-im-seeing-for-my-organization-who-can-i-reach-out-to"></a>J'ai une question spécifique sur les données que je vois pour mon organisation. À qui puis-je m'adresser ?

Vous pouvez utiliser le bouton de rétroaction dans la page d’aperçu de l’activité du centre d’administration, ou vous pouvez ouvrir un [cas de support](../../business-video/get-help-support.md) pour obtenir de l’aide avec l’application modèle. 

### <a name="how-can-partners-access-the-data"></a>Comment des partenaires peuvent-ils accéder aux données ?

Si un partenaire a délégué des droits d’administration, il peut se connecter à l’application modèle au nom de son client.

### <a name="can-i-hide-identifiable-information-such-as-user-group-and-site-names-in-reports"></a>Puis-je masquer des informations d'identification personnelle telles que l'utilisateur, les groupes et les noms de site dans des rapports ?

Oui, voir [Rendre les données collectées anonymes](enable-usage-analytics.md#make-the-collected-data-anonymous).
