# <a name="assign-admin-roles"></a>Attribuer des rôles d’administrateur

Si vous êtes la personne qui a acheté votre abonnement d’entreprise Microsoft, vous êtes l’administrateur mondial. Cela signifie que vous avez un contrôle illimité sur les produits de vos abonnements et que vous pouvez accéder à la plupart des données.

Pour plus d’informations, consultez la rubrique [À propos des rôles d’administrateur](about-admin-roles.md).

Lorsque vous ajoutez de nouveaux utilisateurs, si vous ne leur attribuez pas un rôle d’administrateur, ils sont dans le *rôle de l’utilisateur* et n’ont pas de privilèges d’administration à l’un des centres d’administration Microsoft. Mais si vous avez besoin d’aide pour faire avancer les choses, vous pouvez attribuer un rôle d’administrateur à un utilisateur. Par exemple, si vous avez besoin de quelqu’un pour aider à réinitialiser les mots de passe, vous ne devriez pas leur attribuer le rôle d’administrateur global, vous devez leur attribuer le rôle d’administrateur de mot de passe. Le fait d’avoir un trop grand nombre d’administrateurs généraux, avec un accès illimité à vos données et à votre entreprise en ligne, constitue un risque pour la sécurité.

## <a name="assign-admin-roles"></a>Attribuer des rôles d’administrateur 

::: moniker range="o365-worldwide"

Vous pouvez affecter des utilisateurs à un rôle de 2 façons différentes :

- Vous pouvez aller dans les détails de l’utilisateur et **gérer les rôles** pour attribuer un rôle à l’utilisateur.
- Ou vous pouvez aller à **Rôles** et sélectionner le rôle, puis ajouter plusieurs utilisateurs à elle.

### <a name="assign-admin-roles-to-users-using-roles"></a>Attribuez des rôles d’administrateur aux utilisateurs utilisant des rôles

1. Dans le centre admin, aller à **Rôles**. Choisissez les **onglets Azure AD** **ou Intune** pour afficher les rôles d’administration disponibles pour votre organisation.
2. Sélectionnez le rôle d’administrateur à qui vous souhaitez affecter l’utilisateur.
3. Sélectionnez **Admins** > **Assignés Ajouter**.
4. Tapez le nom d’affichage **ou le nom** **d’utilisateur de** l’utilisateur, puis sélectionnez l’utilisateur dans la liste des suggestions.
5. Ajoutez plusieurs utilisateurs jusqu’à ce que vous avez terminé.
6. Sélectionnez **Enregistrer,** puis l’utilisateur sera ajouté à la liste des administrateurs affectés.

### <a name="assign-a-user-to-an-admin-role-from-active-users"></a>Attribuer un rôle d’administrateur à un utilisateur via l’option Utilisateurs actifs

1. Dans le centre d’administration, allez à **users** > [active users](https://go.microsoft.com/fwlink/p/?linkid=834822) page.

2. Sur la page **Utilisateurs actifs,** sélectionnez l’utilisateur dont vous souhaitez modifier le rôle d’administrateur. Dans le volet flyout, sous **rôles**, sélectionnez **Gérer les rôles**.

3. Sélectionnez le rôle d’administrateur que vous voulez attribuer à l’utilisateur. Si vous ne voyez pas le rôle que vous cherchez, sélectionnez **Afficher le** tout en bas de la liste.

::: moniker-end

::: moniker range="o365-germany"

1. Dans le Centre d’administration, accédez à la page **Utilisateurs** > <a href="https://go.microsoft.com/fwlink/p/?linkid=847686" target="_blank">Utilisateurs actifs</a>.

2. Sur la page **Utilisateurs actifs,** sélectionnez l’utilisateur dont vous souhaitez modifier le rôle d’administrateur. Dans le volet flyout, à côté des **rôles,** sélectionnez **Modifier**. 

    Si vous ne voyez pas l’option **Modifier,** vous n’avez pas la permission de modifier et ne pouvez pas attribuer de rôles d’administrateur à d’autres personnes. Demandez à un administrateur global de votre entreprise d’attribuer des rôles pour vous. Dans une petite entreprise, le propriétaire de l’entreprise (la personne qui a acheté votre abonnement) est un administrateur mondial. Dans une grande entreprise, les personnes clés dans le département informatique sont des administrateurs mondiaux.

3. Sélectionnez **Administrateur** personnalisé pour voir une liste de rôles que nous avons personnalisés pour vous. Pour une description de chaque rôle, voir Sur [les rôles admin.](about-admin-roles.md)

::: moniker-end

::: moniker range="o365-21vianet"

1. Dans le Centre d’administration, accédez à la page **Utilisateurs** > <a href="https://go.microsoft.com/fwlink/p/?linkid=850628" target="_blank">Utilisateurs actifs</a>.

2. Sur la page **Utilisateurs actifs,** sélectionnez l’utilisateur dont vous souhaitez modifier le rôle d’administrateur. Dans le volet flyout, à côté des **rôles,** sélectionnez **Modifier**.

    Si vous ne voyez pas l’option **Modifier,** vous n’avez pas la permission de modifier et ne pouvez pas attribuer de rôles d’administrateur à d’autres personnes. Demandez à un administrateur global de votre entreprise d’attribuer des rôles pour vous. Dans une petite entreprise, le propriétaire de l’entreprise (la personne qui a acheté votre abonnement) est un administrateur mondial. Dans une grande entreprise, les personnes clés dans le département informatique sont des administrateurs mondiaux.

3. Sélectionnez **Administrateur** personnalisé pour voir une liste de rôles que nous avons personnalisés pour vous. Pour une description de chaque rôle, voir Sur [les rôles admin.](about-admin-roles.md)

::: moniker-end

Utilisez les instructions suivantes pour attribuer des rôles à des dizaines d'utilisateurs.

::: moniker range="o365-worldwide"

## <a name="check-admin-roles-in-your-organization"></a>Vérifiez les rôles d’administrateur dans votre organisation

Il se peut que vous n’avez pas les autorisations correctes pour attribuer des rôles d’administrateur à d’autres utilisateurs. Vérifiez si vous avez les autorisations correctes ou demandez à un autre administrateur d’attribuer des rôles pour vous.

Vous pouvez vérifier les autorisations de rôle admin de 2 façons différentes :

- Vous pouvez aller dans les détails de l’utilisateur et regarder sous **rôles** sur la page **Compte.**
- Ou vous pouvez aller dans les **rôles** et sélectionner le rôle d’administrateur, et sélectionner les administrateurs affectés pour voir quels utilisateurs sont affectés.

::: moniker-end
