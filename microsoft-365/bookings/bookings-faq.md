# <a name="microsoft-bookings-frequently-asked-questions"></a>Questions fréquemment posées sur Microsoft Bookings

## <a name="general"></a>Général

### <a name="what-is-microsoft-bookings"></a>Qu’est-ce que Microsoft Bookings ?

Microsoft Bookings est une application Microsoft 365 qui facilite la planification et la gestion des rendez-vous. Bookings inclut un calendrier de réservation basé sur le Web et s’intègre à Outlook pour optimiser le calendrier de votre personnel, offrant ainsi à vos clients la flexibilité nécessaire pour réserver un temps qui leur est le mieux réservé. Les e-mails de notification automatisés réduisent les absences de shows et les organisations gagnent du temps avec une réduction des tâches de planification répétitives. Bookings vous aide à effectuer facilement des rendez-vous virtuels via Skype ou Microsoft Teams, et vous permet de gérer la planification quotidienne via l’application Bookings dans Teams. Avec la possibilité intégrée de personnaliser, Bookings est conçu pour répondre aux besoins de plusieurs parties de n’importe quelle organisation.

### <a name="how-do-i-use-microsoft-bookings"></a>Comment utiliser Microsoft Bookings ?

Bookings est un service en ligne, vous n’avez donc rien à télécharger. Il vous suffit d’aller au sériateur d’application dans l’expérience Web Microsoft 365. Les administrateurs peuvent également utiliser l’application complémentaire Bookings pour rester à jour avec les dernières informations sur les clients et leurs rendez-vous.

### <a name="who-has-access-to-microsoft-bookings"></a>Qui a accès à Microsoft Bookings ?

Bookings est disponible et actif par défaut pour les clients Microsoft 365 Business Premium, Microsoft 365 Business Standard, A3, A5, E3 et E5 dans le monde entier. Bookings est également disponible dans Office 365 géré par 21Vianet.

### <a name="will-my-customers-see-my-personal-or-business-calendar"></a>Mes clients voient-ils mon calendrier personnel ou d’entreprise ?

Vos clients ne voient que le calendrier Bookings que vous publiez en ligne pour les services, les heures et le personnel que vous choisissez d’inscrire.

### <a name="how-do-i-change-a-bookable-attribute-under-manage-staff"></a>Comment modifier un attribut bookable sous Gérer le personnel ?

Lorsque des membres du personnel ont été ajoutés, ils auraient reçu un courrier électronique pour accepter ou rejeter la demande. Ils peuvent ouvrir le même e-mail et sélectionner « Rejeter » pour les rendre non bookables. Nous n’avons actuellement aucun moyen de le faire à partir de l’intérieur de l’application.

### <a name="how-do-end-users-access-the-bookings-app"></a>Comment les utilisateurs finaux accèdent-ils à l’application Bookings ?

Toute personne titulaire d’une licence Microsoft 365 Business Premium, Microsoft 365 Business Standard, A3, A5, E3, E5 ou ajoutée en tant que membre du personnel dans les rôles Administrateur ou Visionneuse peut accéder aux réservations dans le s’il s’agit de l’expérience Web Microsoft 365. Il existe également une application complémentaire disponible pour iOS et Android.

### <a name="can-i-use-bookings-in-my-own-web-site"></a>Puis-je utiliser Bookings dans mon propre site Web ?

Oui. Nous vous fournissons un moyen d’incorporer votre calendrier Bookings dans votre site via un iFrame. Le code d’incorporation de lien se trouve dans **l’onglet de la page Réservations** dans l’application Bookings.

### <a name="can-i-use-bookings-even-if-i-dont-have-a-web-page-for-my-business"></a>Puis-je utiliser Bookings même si je n’ai pas de page Web pour mon entreprise ?

Oui. Nous fournissons un lien vers votre page de réservation dans **l’onglet Page** de réservation dans l’application Web. Il vous suffit de fournir ce lien à vos clients et ils voient les dernières informations de disponibilité de votre entreprise. Vous pouvez également partager la page de réservation sur les réseaux sociaux ou même utiliser la fonctionnalité d’incorporation pour l’héberger dans un iFrame. Vous avez également la possibilité de contrôler qui peut accéder à la page en désactivant l’indexation directe du moteur de recherche et en limitant l’accès à ceux au sein de votre organisation.

### <a name="can-our-it-department-control-whether-end-users-can-access-bookings"></a>Notre service informatique peut-il contrôler si les utilisateurs finaux peuvent accéder à Bookings ?

Bookings est disponible par défaut dans les clients Microsoft 365 Business Premium, Microsoft 365 Business Standard, M365 A3/A5 et E3/E5, mais les administrateurs peuvent le désactiver dans le Centre d’administration Microsoft 365 s’ils le souhaitent. Pour ce faire, [suivez ces instructions.](turn-bookings-on-or-off.md)

Si un client souhaite désactiver l’accès à Bookings pour certaines licences éligibles dans son client, il peut utiliser une stratégie de groupe pour restreindre les licences ou implémenter une stratégie de boîte aux lettres OWA qui limite les utilisateurs autorisés à créer de nouveaux calendriers Bookings.

Si vous utilisez une stratégie de boîte aux lettres OWA pour désactiver l’accès à Bookings, tous les utilisateurs auront une licence Bookings. Toutefois, lorsqu’ils essaieront d’accéder à l’application, ils ne pourront y accéder que s’ils font partie de la stratégie ou s’ils ont été ajoutés à un calendrier Bookings existant en tant que membres du personnel. Pour plus [d’informations,](turn-bookings-on-or-off.md) voir la section « Autoriser uniquement les utilisateurs sélectionnés à créer des calendriers Bookings ».

Lors de la configuration d’un calendrier Bookings, de nombreux aspects de la page de planification web, les informations professionnelles, les détails du personnel, les types de service et les stratégies de planification peuvent être personnalisés.

### <a name="is-all-the-functionality-of-the-original-bookings-web-app-available-in-microsoft-teams"></a>Toutes les fonctionnalités de l’application Web Bookings d’origine sont-elle disponibles dans Microsoft Teams ?

## <a name="pricing-and-licensing"></a>Tarification et gestion des licences

### <a name="how-do-i-get-microsoft-bookings"></a>Comment obtenir Microsoft Bookings ?

Bookings est disponible dans Microsoft 365 pour les clients titulaires de licences Microsoft 365 Business Premium, Microsoft 365 Business Standard, A3, A5, E3 et E5, ainsi qu’une application complémentaire iOS et Android. Bookings n’est pas disponible en tant qu’application autonome. Outlook Web App ou Outlook sur le web doit être activé pour utiliser Bookings, car il stocke des données dans Outlook.

Une licence Bookings fournit des fonctionnalités complètes au produit, notamment la création et la gestion de calendriers. Il permet également aux utilisateurs d’afficher et de modifier des calendriers existants, lorsque ces utilisateurs sont ajoutés en tant que membres du personnel dans un rôle d’administrateur ou de visionneuse.

### <a name="do-i-need-to-migrate-my-email-account-from-my-current-provider-to-microsoft-365"></a>Dois-je migrer mon compte de messagerie de mon fournisseur actuel vers Microsoft 365 ?

Vous pouvez conserver votre fournisseur actuel, mais Bookings enverra toutes les notifications au courrier électronique utilisé dans votre compte Microsoft 365.

### <a name="can-my-employees-use-bookings-without-a-microsoft-365-account"></a>Mes employés peuvent-ils utiliser Bookings sans compte Microsoft 365 ?

Oui. Vous pouvez ajouter votre personnel avec n’importe quel e-mail, et il reçoit toujours la confirmation par courrier électronique et l’invitation du calendrier lorsqu’une personne écrit un rendez-vous avec lui.

### <a name="can-i-create-more-than-one-bookings-calendar-under-the-same-microsoft-365-account-and-switch-between-them"></a>Puis-je créer plusieurs calendriers Bookings sous le même compte Microsoft 365 et basculer entre eux ?

Oui. Vous pouvez créer et gérer plusieurs calendriers Bookings avec un compte. Vous pouvez basculer d’un site à l’autre à l’aide de l’attention en face du nom de l’entreprise dans l’application Web Bookings.

### <a name="what-if-my-organization-has-mixed-licenses-with-f1f3-e1-and-e3e5"></a>Que se passe-t-il si mon organisation possède des licences mixtes avec F1/F3, E1 et E3/E5 ?

Nous savons que de nombreuses organisations ont une combinaison de licences Microsoft 365. Par exemple, un client peut avoir des licences M365 E3 pour les employés dans son siège social, mais des licences M365 E1 (ou F1 ou F3) pour ses employés de magasin.

Dans cet exemple, les employés du siège social titulaires d’une licence M365 E3 ont un accès complet à Bookings, ce qui signifie qu’ils peuvent créer de nouveaux calendriers, modifier des paramètres, ajouter du personnel, publier une page de réservation, créer et gérer des rendez-vous dans le calendrier, et tirer des rapports.

Ces employés du magasin avec des licences E1/F1/F3, ou sans licence, peuvent toujours être ajoutés en tant que membres du personnel aux calendriers dans un rôle invité, puis réservé à des rendez-vous, et ils reçoivent des e-mails de confirmation lorsqu’ils sont bookés. Elles peuvent toujours être réservées pendant leurs heures disponibles/prévues, comme indiqué dans l’onglet Du personnel. La disponibilité du calendrier Bookings limite les heures de réservation en fonction des heures définies et des heures de service.

Les employés du Store s’affichent également comme indisponibles dans Bookings si un rendez-vous a déjà été programmé à ce moment-là via l’application Web Bookings. Les rendez-vous bookés via Bookings seront reflétés comme étant occupés dans le calendrier d’un membre du personnel dans Bookings. Les employés titulaires d’une licence non-bookings peuvent toujours avoir un impact sur leur calendrier personnel dans Bookings, à condition qu’ils se trouveront dans le même client.

Les personnes dans le rôle Invités peuvent afficher les informations fournies par le client lors de la création initiale du rendez-vous. Par exemple, si le représentant du magasin qui est réservé au rendez-vous doit appeler le client avant le rendez-vous, il aura accès aux informations qu’il fournit pendant le processus de planification. Le membre du personnel qui est réservé aura accès à toutes les informations qui apparaissent dans l’e-mail de confirmation, ainsi qu’à l’événement de calendrier .ics (par exemple, le numéro de téléphone du client s’il a été entré).

Les personnes qui jouent le rôle d’invité n’ont pas la possibilité d’accéder à l’application Web Bookings pour modifier les paramètres ou pour afficher et gérer les rendez-vous (ajouter, annuler et reprogrammer). Toutefois, ils peuvent prendre des rendez-vous pour le compte de clients à l’aide de la page libre-service, de la même façon qu’un client effectuerait un rendez-vous.

Nous vous recommandons de mettre sous licence un responsable commercial ou un administrateur de chaque magasin avec une licence éligible à Bookings pour configurer et gérer des pages et des rendez-vous. Le reste du personnel travaillerait ensuite avec l’employé titulaire d’une licence Bookings afin de reprogrammer ou d’annuler une réservation.

## <a name="product-features"></a>Fonctionnalités du produit

### <a name="where-do-bookings-calendars-show-up-in-my-microsoft-365-tenant"></a>Où les calendriers Bookings s’affiche-t-ils dans mon client Microsoft 365 ?**

Chaque nouveau calendrier Bookings crée une boîte aux lettres correspondante dans Exchange, ainsi qu’une entrée associée dans Azure Active Directory (AAD), où l’entrée est répertoriée en tant qu’utilisateur sans permis.

### <a name="can-i-delete-a-previously-created-bookings-calendar"></a>Puis-je supprimer un calendrier Bookings créé précédemment ?

Pour supprimer un calendrier Bookings, vous devez supprimer la boîte aux lettres associée dans Exchange.

### <a name="if-i-create-a-calendar-and-someone-else-has-access-to-bookings-would-they-be-able-to-see-my-calendar"></a>Si je crée un calendrier et que quelqu’un d’autre a accès à Bookings, est-ce qu’il peut voir mon calendrier ?

Les seules personnes qui ont accès aux calendriers Bookings que vous créez (via l’application Web) sont les personnes qui ont été ajoutées en tant que membres du personnel, en tant qu’administrateur ou visionneuse. Les administrateurs client pourront voir la liste de toutes les boîtes aux lettres Bookings dans Exchange et AAD.

### <a name="are-video-conferencing-meetings-integrated-into-the-bookings-app"></a>Les réunions de vidéoconférence sont-elles intégrées à l’application Bookings ?

Les réunions en ligne sont disponibles dans Bookings lorsque vous utilisez les clients Skype ou Microsoft Teams. Les réunions en ligne peuvent être activées au niveau du service, et le client de réunion que vous choisissez (Skype ou Teams) sera celui qui est définie par défaut pour un membre du personnel réservé. Pour participer aux réunions en ligne, chaque nouveau rendez-vous aura un lien de réunion unique joint au rendez-vous, et en outre, des options de rejoindre facilement l’événement sont incluses dans l’événement et dans les e-mails de confirmation.

### <a name="how-does-scheduling-policy-work"></a>Comment fonctionne la stratégie de planification ?

Un paramètre appelé **Durée maximale** détermine le plus à l’avance (mesuré en jours) qu’une réservation peut être réalisée. Il doit y avoir au moins 24 heures entre le temps d’avance minimal **(durée** minimale, en heures, pour les réservations et les annulations) et le temps d’avance maximal. L’heure d’avance maximale peut être définie sur un jour si l’heure d’avance minimale est définie sur 0 heure, ce qui signifie dans ce cas qu’un client peut annuler une réservation jusqu’à son démarrage, et que les clients ne peuvent planifier les réservations que si le rendez-vous se trouve dans le jour suivant.

### <a name="how-does-bookings-work-across-time-zones"></a>Comment fonctionne Bookings sur plusieurs fuseaux horaires ?

Toutes les heures sont dans le fuseau horaire d’entreprise (votre fuseau horaire local) par défaut. Cela signifie que tous les paramètres que vous configurez pour un calendrier Bookings, tels que les heures de travail, apparaîtront dans ce fuseau horaire. La page libre-service a la possibilité d’afficher toutes les heures de rendez-vous dans le fuseau horaire de l’utilisateur final, qui peuvent être désactivées si vous le souhaitez. Si **toujours** afficher les créneaux horaires dans le fuseau horaire d’entreprise reste désactivé sous l’onglet de la page Bookings, les personnes qui visitent la page voient les créneaux horaires dans leurs propres fuseaux horaires locaux.

:::image type="content" source="media/bookings-faq-region.png" alt-text="Paramètres de région et de fuseau horaire Bookings":::

Il n’est pas prévu de définir de fuseau horaire pour le personnel dans Bookings. Le fuseau horaire du personnel, et donc les heures d’ouverture, sera dans le fuseau horaire d’entreprise.

### <a name="can-email-notifications-be-sent-as-the-tenant-domain-name"></a>Des notifications par courrier électronique peuvent-ils être envoyées en tant que nom de domaine client ?

Les adresses de messagerie sont contrôlées et gérées au niveau des paramètres Microsoft 365 et dépendent des paramètres de configuration de domaine qui y sont employés. Vous pouvez trouver plus d’informations [ici.](/powershell/module/exchange/get-accepteddomain)

### <a name="is-the-ability-to-send-sms-messages-dependent-on-a-different-service-or-specific-configuration"></a>La possibilité d’envoyer des messages SMS dépend-elle d’un service différent ou d’une configuration spécifique ?

Les messages SMS sont actuellement disponibles en Amérique du Nord et un compte Skype ou Twilio est utilisé pour la remise des SMS.

### <a name="how-can-bookings-appointments-show-up-on-a-persons-calendar"></a>Comment les rendez-vous Bookings peuvent-ils s’afficher dans le calendrier d’une personne ?

La confirmation de la réservation est envoyée à la boîte de réception du fournisseur de services et du client. L’e-mail de confirmation contient une pièce jointe *.ics, qui peut ensuite être ajoutée au calendrier de l’utilisateur avec tous les détails de rendez-vous pertinents.

### <a name="what-triggers-service-provider-or-staff-and-customer-emails"></a>Qu’est-ce qui déclenche les messages électroniques du fournisseur de services, du personnel et des clients ?

Les e-mails sont déclenchés en fonction des paramètres de l’onglet Services de l’application Web Bookings. Une réservation réalisée par le client sur la page libre-service ou sous l’onglet Calendrier de l’application Web Bookings déclenche une confirmation et/ou un message électronique de rappel. La même chose se produit lorsqu’une personne  modifie la réservation via le bouton Gérer la réservation dans l’e-mail de confirmation ou dans l’application Web. Les courriers électroniques de rappel sont envoyés à une période spécifiée avant un rendez-vous, comme indiqué dans l’onglet Services de l’application Web Bookings.

### <a name="can-i-book-classroom-style-appointments-that-are-1many-instead-of-11"></a>Puis-je réserver des rendez-vous de type salle de classe qui sont 1:1 au lieu de 1:1 ?

Oui, nous avons une fonctionnalité de réservation de groupe qui permet à plusieurs personnes de réserver le même rendez-vous en même temps (par exemple, pour un cours de mise en forme). Cette fonctionnalité est décrite en détail [ici.](https://techcommunity.microsoft.com/t5/microsoft-bookings-blog/microsoft-bookings-now-supports-online-meetings-and-group/ba-p/1214120)

### <a name="can-calendars-remain-unpublished-not-public-facing-but-still-accessible-to-designated-users"></a>Les calendriers peuvent-ils rester non publiés (non publics) mais toujours accessibles aux utilisateurs désignés ?

La sélection de cette case à cocher limite l’accès aux pages libre-service uniquement à celles qui se trouve dans votre client. Le calendrier Bookings qui se trouve dans l’application Web pour la création et la gestion est accessible uniquement par les personnes ajoutées en tant que membres du personnel à la page avec des rôles d’administrateur ou de visionneuse.

Le processus serait d’utiliser l’API Graph pour obtenir les détails de la boîte aux lettres, puis d’utiliser ces détails pour créer une nouvelle boîte aux lettres. La documentation de l’API [est ici.](/graph/api/resources/booking-api-overview?view=graph-rest-beta)

Tous les membres du personnel Bookings affectés au rôle Administrateur peuvent télécharger des valeurs séparées par des onglets (. Fichier TSV avec toutes les réservations réalisées au cours des 120 derniers jours. Les rapports Bookings peuvent être téléchargés dans . Formulaire TSV de l’onglet Accueil de la page Bookings. Oui, vous pouvez créer des calendriers Bookings distincts pour chaque service, ou vous pouvez accéder à l’onglet Services, modifier un service et, en haut de la page, une option vous permet de partager une URL uniquement pour ce service spécifique.

### <a name="what-options-are-available-for-embedding-a-consent-form-or-disclosure-for-those-booking-an-appointment"></a>Quelles options sont disponibles pour incorporer un formulaire de consentement ou une divulgation pour les personnes qui ont rendez-vous ?

Nous vous  recommandons d’activer la case à cocher de consentement d’utilisation des données client dans la page Réservation et de personnaliser la formulation pour transmettre correctement les exigences de divulgation ou de consentement de votre organisation. Une autre option consiste à ajouter au service un champ personnalisé qui inclut un lien vers le formulaire de consentement et à demander aux clients de confirmer qu’ils ont rempli le formulaire avant de pouvoir passer à un rendez-vous. Vous pouvez également ajouter l’URL du formulaire de consentement sous forme de notes supplémentaires dans les e-mails de confirmation et de rappel, mais cela n’empêche pas les utilisateurs de prendre un rendez-vous.

### <a name="what-changes-can-be-made-to-the-staff-selection-capability-on-the-self-service-booking-page"></a>Quelles modifications peuvent être apportées à la fonctionnalité de sélection du personnel sur la page de réservation en libre-service ?

La possibilité pour les clients de sélectionner des membres du personnel  spécifiques lorsque la réservation peut être supprimée complètement en désélecrant l’option Autoriser les clients à choisir une personne spécifique pour l’option de réservation dans la page réservation ou la section Services. Les réservations affectent automatiquement un membre du personnel disponible de manière aléatoire à la réservation, en fonction de la disponibilité du personnel.

### <a name="can-bookings-support-many-concurrent-booking-requests-and-self-service-page-visits"></a>Bookings peut-il prendre en charge de nombreuses demandes de réservation simultanées et des visites de page en libre-service ?

Booking peut prendre en charge un grand nombre de visiteurs et de réservations à la fois. Si la page reçoit un très grand volume de trafic, les utilisateurs reçoivent une erreur de « serveur occupé ». La disponibilité des rendez-vous est mise à jour lors du chargement de la page, ainsi que lors d’une réservation. Si plusieurs personnes tentent de réserver le même rendez-vous à la fois, Bookings ne laisse qu’une seule personne réserver cette heure et envoie un message de notification aux autres clients potentiels et les invite à trouver une autre heure.

## <a name="privacy"></a>Confidentialité

### <a name="where-is-bookings-data-stored"></a>Où les données Bookings sont-elles stockées ?

Bookings est une application Microsoft 365, ce qui signifie que toutes les données sont stockées dans la plateforme Microsoft 365 et dans Exchange. Bookings suit toutes les stratégies de stockage de données définies par Microsoft, qui sont les mêmes stratégies suivies par toutes les applications Office. Bookings utilise des boîtes aux lettres partagées dans Exchange pour stocker les détails du client, du personnel, du service et des rendez-vous. Les stratégies de conformité pour les boîtes aux lettres partagées dans Exchange s’appliquent également aux boîtes aux lettres Bookings. Toutes les données client (y compris les informations fournies par les clients lors de la réservation) sont capturées dans Bookings et stockées dans l’application, elles sont donc stockées dans Exchange. Microsoft Bookings utilise les mêmes stratégies que microsoft 365, que vous trouverez [ici.](https://www.microsoft.com/online/legal/v2/?docid=22&langid=en-us)

### <a name="is-there-a-way-to-centrally-manage-bookings-calendars-for-all-users"></a>Existe-t-il un moyen de gérer de manière centralisée les calendriers Bookings pour tous les utilisateurs ?

Chaque calendrier Bookings est conservé indépendamment. Il n’existe aucune vue consolidée, autre que la zone de la zone de recherche après avoir cliqué sur le point d’accueil de la page d’accueil.

### <a name="how-are-users-authenticated"></a>Comment les utilisateurs sont-ils authentifiés ?

L’accès à l’application Web Bookings implique l’authentification via l’authentification Azure Active Directory (AAD) normale. La page de réservation en libre-service peut être accessible à tout le monde avec le lien de page Web. Toutefois, lorsque le paramètre Exiger un compte **Microsoft 365 ou Office 365** de mon organisation pour réserver est sélectionné, la page est limitée pour une utilisation uniquement dans le client Microsoft 365 (à l’aide de l’authentification AAD).

:::image type="content" source="media/bookings-faq-access-ctrl.png" alt-text="Contrôle d’accès à la page de réservation dans Bookings":::

### <a name="does-customer-data-leave-the-production-system-for-any-reason"></a>Les données client quittent-elles le système de production pour une raison quelconque ?

Oui, étant donné que Bookings fait partie d’Exchange, nous permettons aux API Graph et aux API des services web Exchange (EWS) pour les clients de télécharger leurs propres données à laquelle ils ont accès.

### <a name="is-there-the-ability-to-customize-email-addresses-for-customer-replies"></a>Y a-t-il la possibilité de personnaliser les adresses de messagerie pour les réponses des clients ?

Il est possible de définir l’adresse de messagerie à laquelle le client envoie ses réponses. Ce paramètre se trouve sous **l’onglet Informations** professionnelles. Le nom de la boîte aux lettres Bookings lui-même peut également être modifié dans Exchange, par le biais de processus de changement de nom de boîte aux lettres Exchange habituels.

### <a name="we-would-like-for-the-customer-to-accept-our-terms-and-conditions-during-the-booking-process-is-that-possible"></a>Nous voulons que le client accepte nos conditions générales pendant le processus de réservation. Est-ce possible ?

Cette fonctionnalité existe déjà pour la conformité au R GDPR. La page Booking contient un champ pour le consentement personnalisé, juste en dessous de la section détails de publication. Il vous suffit de cocher cette case et d’écrire votre propre message de consentement personnalisé qui s’affichera en tant que champ obligatoire sur la page libre-service. La date et l’heure à laquelle le client accorde son consentement seront enregistrées.

Cette section ne prend pas en charge le texte enrichi, mais pensez que des liens peuvent être ajoutés sous la section Informations professionnelles et s’afficheraient juste en dessous de la case à cocher de consentement.

### <a name="how-can-bookings-associated-mailboxes-in-exchange-be-identified"></a>Comment les boîtes aux lettres associées à Bookings dans Exchange peuvent-ils être identifiées ?

La commande suivante permet de récupérer les informations de toutes les boîtes aux lettres Bookings et des utilisateurs qui y ont accès.

`Get-Mailbox -RecipientTypeDetails SchedulingMailbox -ResultSize:Unlimited | Get-MailboxPermission |Select-Object Identity,User,AccessRights | Where-Object {($_.user -like '@')}`

## <a name="technical-questions"></a>Questions techniques

### <a name="if-a-user-schedules-an-appointment-for-a-service-consisting-of-an-online-meeting-how-do-they-receive-the-microsoft-teams-link-to-the-meeting-if-they-leave-the-optional-email-address-field-empty"></a>Si un utilisateur planifiera un rendez-vous pour un service constitué d’une réunion en ligne, comment recevoir le lien Microsoft Teams vers la réunion s’il laisse le champ d’adresse de messagerie facultatif vide ?

Si le système ne comprend pas les informations de courrier du client, il n’existe aucun moyen de remettre le message de confirmation au client, et par conséquent, il ne recevra pas de lien Teams. La solution consiste à définir l’adresse de messagerie comme champ obligatoire, ce que vous pouvez faire sous l’onglet Services de Bookings. Notez que chaque fois que vous créez un service dans Bookings, le champ courrier du client est définie comme champ obligatoire par défaut.

### <a name="how-are-staff-assigned-to-appointments-when-anyone-is-selected-or-when-customers-do-not-have-the-ability-to-choose-a-staff-member-for-the-appointment"></a>Comment le personnel est-il affecté aux rendez-vous lorsque « Tout le monde » est sélectionné ou lorsque les clients n’ont pas la possibilité de choisir un membre du personnel pour le rendez-vous ?

Le personnel est affecté de manière aléatoire dans ces scénarios, en dehors de tous les membres du personnel disponibles au moment de la réservation du rendez-vous. Il n’existe actuellement aucun moyen de garantir une répartition égale des rendez-vous affectés de manière aléatoire entre tous les membres du personnel, ni de chiffrer le nombre de rendez-vous affectés à chaque membre du personnel. Si un équilibrage de charge doit être effectué au sein de votre personnel après un certain nombre d’affectations aléatoires, il peut être effectué manuellement dans l’onglet Calendrier de la page Réservations.

### <a name="how-can-users-opt-out-of-receiving-sms-messages-if-they-have-already-opted-in-previously"></a>Comment les utilisateurs peuvent-ils refuser de recevoir des messages SMS s’ils l’ont déjà fait ?

Lorsque vous rendez-vous avec des SMS activés, les clients sont invités à cocher la case à cocher chaque fois qu’ils signent. Les clients peuvent également répondre directement au message SMS (le fournisseur de SMS) indiquant qu’ils souhaitent que les messages cessent, puis aucun texte supplémentaire n’est envoyé à ce numéro de téléphone. Si le client ne reçoit plus de sms du fournisseur, et non de Bookings lui-même, il ne recevra plus de MESSAGES SMS à l’avenir, même si vous avez choisi de le faire à partir de la page Bookings.

## <a name="troubleshooting"></a>Résolution des problèmes

### <a name="this-staff-members-calendar-cant-be-synced"></a>Le calendrier de ce membre du personnel ne peut pas être synchronisé

Si vous recevez l’erreur « Le calendrier de ce membre du personnel ne peut pas être synchronisé », cette section contient plusieurs procédures de dépannage pour vous aider à résoudre l’erreur.

> [!NOTE]
> Les réservations sont désactivées par défaut pour les clients qui ont des abonnements Microsoft 365 Business Standard, Microsoft 365 A3 ou Microsoft 365 A5. Bookings est également disponible pour les clients qui ont Office 365 Entreprise E3 et Office 365 Entreprise E5, mais il est désactivé par défaut. To get started, see [Get access to Microsoft Bookings](get-access.md). Pour activer ou désactiver bookings, voir Activer ou [désactiver bookings pour votre organisation.](turn-bookings-on-or-off.md)

Si vous recherchez des informations sur la synchronisation du calendrier de votre personnel, voir [Ajouter du personnel à Bookings.](add-staff.md) Dans la page **Détails** du personnel, les événements **du calendrier affectent la disponibilité** sont vérifiés.

### <a name="the-staff-member-needs-to-change-their-calendar-permissions"></a>Le membre du personnel doit modifier ses autorisations de calendrier

Pour que Bookings soit correctement synchronisé avec les calendriers des membres de votre personnel, chaque membre du personnel doit partager son calendrier. Chaque membre du personnel doit suivre ces instructions pour partager son propre calendrier.

1. Connectez-vous à Microsoft 365, puis sélectionnez **Outlook** dans le lanceur d’applications.

   > [!div class="mx-imgBorder"]
   > ![Image d’Outlook dans le lanceur d’applications](../media/bookings-applauncher-outlook.png)

1. En haut de la page, sélectionnez **Partager le** \> **calendrier.**

   ![Image du menu de partage de calendrier](../media/bookings-share-calendar.png)

1. Dans la section **Personnes à l’intérieur** de votre organisation, sélectionnez la zone de baisse en regard de Mon **organisation,** puis sélectionnez Peut afficher lorsque **je suis occupé.**

   > [!NOTE]
   > Ce paramètre partage vos informations de libre-service avec Bookings et avec d’autres utilisateurs de votre organisation. Il n’inclut pas d’autres informations, telles que le type de rendez-vous que vous avez programmé, les personnes avec qui vous avez programmé le rendez-vous ou l’emplacement.

   > [!div class="mx-imgBorder"]
   > ![Image de l’écran de partage de calendrier avec Peut afficher lorsque je suis occupé](../media/bookings-view-busy.png)

1. Sélectionnez **Terminé**.

### <a name="we-cant-find-a-staff-member-in-our-organization"></a>Nous ne pouvons pas trouver de membre du personnel dans notre organisation

Si un employé a quitté votre entreprise et que vous l’avez supprimé de votre organisation Microsoft 365 ou Office 365, vous devez toujours supprimer cet utilisateur de Bookings afin qu’il ne s’affiche pas dans le calendrier et pour empêcher les clients de prendre des rendez-vous avec eux.

1. Connectez-vous à Microsoft 365, puis **rendez-vous** sur Bookings \> **Staff**.

1. Placez le curseur de la souris sur le nom du membre du personnel. **Les icônes** **Modifier** et Supprimer apparaissent à droite de la **colonne** Téléphone.

1. Sélectionnez **l’icône** Supprimer.

1. Dans la zone de confirmation, sélectionnez **OK.**

> [!NOTE]
> Si vous avez plusieurs calendriers de réservation, vous devez supprimer l’utilisateur de tous ces calendriers.

Pour vérifier si vous avez plusieurs calendriers, dans le menu de gauche, sélectionnez la flèche déroulante en regard du nom de votre société, puis sélectionnez **Ouvrir**. Si vous n’avez qu’un seul calendrier, vous ne verrez que l’option d’ajout d’un calendrier de réservation. Cet exemple montre qu’il existe un deuxième calendrier nommé « Contoso CO ».

![Image de l’écran Choisir un calendrier avec un deuxième calendrier affiché](../media/bookings-choose-calendar.png)

### <a name="theres-a-syncing-error-we-cant-identify"></a>Erreur de synchronisation que nous ne pouvons pas identifier

Il s’agit d’une erreur temporaire qui se produit généralement en raison d’une connexion Internet instable. Ces problèmes se résolvent généralement eux-mêmes après quelques minutes.

Si vous voyez toujours cette erreur après une période prolongée ou si vous la voyez souvent, contactez le support technique.
